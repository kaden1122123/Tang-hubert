### Discord Delivery Fix (continued from 04-30)

**Root cause:**
- Cron job uses `announce -> discord:channel:1485058398486532167` (explicit channel ID) → delivery success
- Interactive Discord session has `deliveryContext.channel = "discord"` (generic string, no channel ID) → reply fails
- Session stuck in `processing` (76k tokens) → no final reply generated
- `messages.groupChat.visibleReplies` defaults to `manual`

**Solution:**
1. Add `"groupChat": {"visibleReplies": "automatic"}` in messages block
2. Restart gateway to clear stuck sessions
3. Hubert tested in another Discord channel — works ✅

**Key documentation finding:**
> "In guild channels, normal assistant final replies stay private by default. Visible Discord output must be sent explicitly with the `message` tool"
> "To restore legacy automatic final replies for group/channel rooms, set `messages.groupChat.visibleReplies: "automatic"`."
