# 2026-05-05

## What I Learned
- Threads API uses `graph.threads.net/v1.0/` (different from Instagram's `graph.facebook.com`)
- Threads API requires `multipart/form-data`, not JSON body
- Two-step publish: `POST /threads` (create container) → `POST /threads_publish` (actually publish)
- `status: FINISHED` means draft container ready, NOT published on platform

## Notes

### Threads API Integration — Success

**Problem**: Hubert wanted to auto-post from brtclaw to @ien_vision via Meta Threads API

**Pitfalls:**

**Stage 1: Token obtained but unusable**
- `.env` had `META_INSTAGRAM_API_KEY` (Instagram token) + new `THREADS_ACCESS_TOKEN` (Threads token)
- Threads token couldn't post successfully at first

**Stage 2: Wrong API Host**
- Instagram/FB Graph API: `graph.facebook.com`
- Threads API: `graph.threads.net/v1.0/` — different hosts caused failures

**Stage 3: Container created but never published**
- `POST /{threads_user_id}/threads` → got `container_id`
- API returned `{"id":"...","status":"FINISHED"}`
- **Misconception**: `FINISHED` = published, but actually just "draft container media processing complete"
- Accumulated 3 orphan containers, none became actual posts

**Stage 4: Two-step flow (correct)**
```
Step 1: POST /{threads_user_id}/threads
        → Create draft, return container_id
        Body (multipart/form-data):
        - text={copy}
        - media_type=text
        - access_token={TOKEN}

Step 2: POST /{threads_user_id}/threads_publish
        → Actually publish, return real post_id
        Body (multipart/form-data):
        - creation_id={container_id}
        - access_token={TOKEN}
```

**Final success:**
- Post ID: `18123782929614701`
- Permalink: https://threads.net/@ien_vision/post/18123782929614701
- Username: @ien_vision
- Threads User ID: `27247241604868629`

### Key Learnings

1. **Different API hosts:**
   - Instagram/FB: `graph.facebook.com`
   - Threads: `graph.threads.net/v1.0/`

2. **HTTP Method:**
   - Threads API requires `multipart/form-data`, not JSON body

3. **Two-step publish flow:**
   - `POST /threads` → create container
   - `POST /threads_publish` → actually publish (cannot skip this step)

4. **Don't be misled by `status: FINISHED`:**
   - `FINISHED` ≠ "published on platform"
   - Just means "draft container media processing complete"
