### multi-agent-architecture-discussion.md Key Points

(Hubert × brtclaw × brthermes evolution conclusion)

**Bottleneck diagnosis**: Problem is NOT "not enough agents" but "no convergence mechanism"
- Symptom: system prompt bloat → consensus lost, too many skills → direction drifts, combined effect unknown → no feedback signal

**Three-layer convergence framework:**
- Spec: Each agent aligns before action (Planner maintains)
- Verify: Objective feedback (tests, type check, Lint)
- Score: Quantitative feedback

**Architecture:**
```
Planner (brtclaw) ← Spec + final review
        ↓
Multi-Agent Core: Executor + Critic paired
        ↓
Verifier (automated tests)
        ↓
Planner review (has veto)
```

**Key design principles:**
1. Critic must NOT be same model as Executor
2. Verifier should be automated (critical)
3. Planner retains veto power
