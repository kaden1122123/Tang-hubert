### Discord Reply Problem

**Symptoms:**
- Cron jobs output to Discord correctly
- User mentions bot (@brt1122claw), bot receives and processes, but reply doesn't reach Discord
- Session stuck in `processing` state

**Root causes:**
1. **Stuck sessions**: Multiple Discord channel sessions (`1484900210856693851`, `1486451416431071262`) stuck in `running` state, occupying main agent
2. **Cognee timeout加重**: `before_prompt_build` handler timeout (15000ms), causing session processing delay
3. **Event Loop Delay**: eventLoopDelayMaxMs reaches 13388ms, CPU core ratio 1.09 — severe blockage

**Fix applied:**
```json
{
  "messages": {
    "ackReactionScope": "group-mentions",
    "groupChat": {
      "visibleReplies": "automatic"
    }
  }
}
```
`systemctl --user stop openclaw-gateway` then `start openclaw-gateway`

**Verification**: Hubert tested in another Discord channel — success ✅
